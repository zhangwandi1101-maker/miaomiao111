# 喵喵外企求职利器

一句话项目简介：一个面向外企求职者的简历驱动型岗位推荐 Skill，帮助用户从简历画像出发，筛选、排序并生成可视化的外企岗位投递报告。

## 解决了谁的问题（PRD 浓缩版）

本项目主要服务想投递外企实习、校招、初级岗位或跳槽机会的学生和职场人，尤其是“知道想进外企，但不知道投什么、去哪投、怎么判断匹配度”的用户。

用户上传或粘贴简历后，Skill 会先抽取教育背景、专业、毕业时间、语言能力、实习经历、项目经历、技能关键词和岗位偏好，再结合目标城市、行业、岗位方向、到岗时间等信息，从公开可验证的外企岗位来源中检索机会。最终输出岗位匹配排序、推荐理由、风险提醒、简历修改建议和原始投递链接，减少用户在官网、招聘平台和社媒推荐之间反复搜索、对比和判断的时间成本。

项目不会编造岗位、薪资、福利、HR 回复率或录用概率；没有可靠来源或原始链接的岗位不应进入推荐结果。

## 核心功能

1. **简历画像解析**：从简历中提取学历、专业、毕业时间、语言能力、实习/项目经历、证书、技能工具和潜在适配岗位方向。
2. **外企岗位检索与排序**：优先使用公司官网 Careers 页面，其次参考 LinkedIn、猎聘、智联、实习平台等公开可验证来源，并按简历/JD 匹配度、岗位新鲜度、来源可信度、信息完整度、投递路径效率和行业匹配度排序。
3. **可视化投递报告生成**：输出 Top 岗位推荐看板，每个岗位包含公司、岗位、城市、行业、岗位类型、匹配分、优先级、原始投递链接、推荐理由、风险提醒和简历优化建议。

## 技术栈说明

- **项目形态**：Codex/ChatGPT Skill + 静态 HTML 报告页面。
- **核心文件**：
  - `skill.md`：定义 Skill 的使用场景、用户画像字段、岗位检索流程、评分排序规则和输出格式。
  - `resources/website_skill_prompt.md`：定义报告网页的内容结构、字段保留/删除规则和岗位卡片布局。
  - `resources/fronted_design.md`：定义前端视觉风格要求，用于生成更清晰、可读、聚焦的报告页面。
  - `foreign_internship_report.html`：当前静态 Demo 页面。
- **使用的模型**：项目本身不绑定固定模型 API；实际推理、简历解析、岗位理解和报告生成由 Codex/ChatGPT 宿主环境中的大语言模型完成。
- **Prompt 策略**：
  - **Skill Prompt 分层**：用 `skill.md` 约束任务边界、输入信息、检索流程和输出内容，保证推荐围绕“简历 + 外企岗位匹配”展开。
  - **结构化抽取**：先把简历转成候选人画像，再进入岗位检索和评分，避免直接给泛泛职业建议。
  - **可验证来源优先**：要求保留原始链接和检索时间，不编造缺失字段；优先推荐公司官网投递渠道。
  - **评分排序策略**：先做城市、岗位类型、有效链接、到岗时间等硬性筛选，再按匹配度、新鲜度、可信度、完整度、投递效率和行业匹配度排序。
  - **网页生成 Prompt**：用 `resources/website_skill_prompt.md` 和 `resources/fronted_design.md` 控制页面信息层级，只保留岗位推荐卡片，减少咨询报告式冗余内容。
