# 外企求职报告网页修改提示词

请继续修改当前外企求职报告网页，使页面更加简洁、聚焦岗位推荐本身。
目标是让用户打开网页后，只看到清爽的岗位推荐卡片，而不是大量解释性信息或咨询报告式板块。

## 一、整体目标

当前网页已经是单页 HTML 外企求职报告。
请在现有页面结构基础上继续精简信息层级，删除冗余字段和底部总结模块，让页面重点聚焦：

1. 推荐了哪些岗位
2. 每个岗位的公司、岗位、城市、行业和岗位类型
3. 每个岗位的匹配分数和优先级
4. 为什么推荐这个岗位
5. 投递这个岗位有什么风险
6. 简历针对这个岗位应该怎么改

不要把页面做成完整咨询报告，也不要展示过多后台分析过程。

---

## 二、顶部 Header 要求

网页顶部只保留一个标题：

外企求职报告

要求：

1. 标题区域使用浅蓝色背景。
2. 标题之外不要出现任何副标题、说明文字、生成时间、候选人姓名或需求描述。
3. 不要出现 “Meow Career 外企求职推荐看板”。
4. 不要出现“基于某某简历生成”“岗位状态可能变化”等说明。
5. Header 高度不要太高，保持干净、简洁。
6. Header 下方直接进入 Top 岗位推荐看板。

---

## 三、Top 岗位推荐看板保留

页面主体只保留一个核心模块：

Top 岗位推荐看板

该模块展示 Top 岗位卡片。
每个岗位使用独立卡片展示，卡片之间要有明显间距。
卡片整体风格保持白色背景、圆角、轻微阴影、浅色边框、清晰留白。

---

## 四、岗位卡片中需要保留的字段

每张岗位卡片只保留以下字段：

1. 排名
   例如：#1、#2、#3

2. 岗位名称
   例如：Data Analysis Intern、Business Analyst Intern、Product Operations Intern

3. 公司名称
   例如：Deloitte China、Philips、BASF、Roche、SAP 等

4. 匹配分数
   例如：94 匹配分
   匹配分数需要醒目显示，可以放在卡片右上角。

5. 岗位类型
   例如：实习、校招、初级岗位、Project Intern、Part-time Intern

6. 城市
   例如：上海、杭州、宁波、苏州、北京、深圳、远程等

7. 行业
   例如：专业服务、消费品、医疗健康、互联网、制造业、咨询、金融科技等

8. 优先级
   例如：强烈推荐、推荐、可尝试、谨慎投递

9. 原始投递链接
   显示为按钮或文字链接，例如：打开岗位页面

10. 推荐理由
    说明为什么这个岗位适合用户。

11. 风险提醒
    说明投递该岗位前需要注意什么。

12. 简历优化建议
    说明为了投递该岗位，用户简历应该怎么改。

---

## 五、岗位卡片中必须删除的字段

请从每一张岗位卡片中删除以下字段及对应内容：

1. 来源平台
2. 推荐投递渠道
3. 发布/更新时间
4. 岗位匹配点
5. 主要差距
6. 下一步
7. 下一步行动
8. 建议操作
9. 准备材料
10. 是否需要 Cover Letter
11. 投递行动建议

删除后不要留下空白框、空标题或占位符。

也就是说，岗位卡片不要再展示类似下面这些小格子：

* 来源平台：Prosple + Deloitte Careers
* 推荐投递渠道：Prosple 页面跳转雇主系统
* 发布/更新时间：Ongoing
* 简历匹配点：SQL、Snowflake、Excel……
* 主要差距：岗位要求至少 4 个月……

这些内容全部删除。

如果这些信息仍然需要用于内部判断，可以保留在代码数据结构里，但不要渲染到网页上。

---

## 六、岗位卡片推荐布局

每张岗位卡片建议采用以下结构：

### 1. 卡片顶部

左侧显示：

* 排名标签
* 岗位名称
* 公司名称

右侧显示：

* 匹配分数

示例：

#1
Data Analysis and Smart Product Project Intern
Deloitte China

右上角：

94
匹配分

### 2. 基础信息区

用 4 个小信息块展示：

* 岗位类型
* 城市
* 行业
* 优先级

不要再展示来源平台、推荐投递渠道、发布时间。

### 3. 投递链接

放一个清晰按钮：

打开岗位页面

按钮点击后跳转到原始投递链接。
如果没有原始链接，则该岗位不应被展示。

### 4. 推荐理由 / 风险提醒 / 简历优化建议

卡片下半部分只保留三个内容块：

1. 推荐理由
2. 风险提醒
3. 简历优化建议

这三个内容块可以使用三列布局，也可以在移动端自动变成单列。

每个内容块要有标题和正文。
正文要具体、简洁、可执行，不要写成很长的段落。

---

## 七、删除底部模块

请删除页面最下方的以下两个大板块：

1. 岗位风险提醒
2. 简历优化建议

也就是说，页面底部不要再出现全局总结性质的风险提醒卡片，也不要再出现全局简历优化建议卡片。

风险提醒和简历优化建议只保留在每个岗位卡片内部，不再作为页面底部独立模块展示。

删除后页面结构应变成：

1. Header：外企求职报告
2. Top 岗位推荐看板
3. 页面结束

不要再有其他底部总结模块。

---

## 八、页面整体结构

最终页面结构应为：

```html
<header>
  <h1>外企求职报告</h1>
</header>

<main>
  <section class="job-board">
    <h2>Top 岗位推荐看板</h2>

    <article class="job-card">
      <!-- 岗位卡片内容 -->
    </article>

    <article class="job-card">
      <!-- 岗位卡片内容 -->
    </article>
  </section>
</main>
```

不要出现以下 section：

```html
<section class="candidate-profile">
<section class="scoring-methodology">
<section class="application-plan">
<section class="source-confidence">
<section class="missing-info">
<section class="global-risk-warning">
<section class="global-resume-suggestions">
```

---

## 九、视觉风格要求

请保持页面清爽、现代、轻量化。

视觉要求：

1. 页面背景使用浅灰色或非常浅的蓝灰色。
2. Header 使用浅蓝色。
3. 岗位卡片使用白色背景。
4. 卡片圆角建议 12px 到 18px。
5. 卡片边框使用浅蓝灰色。
6. 卡片阴影要轻，不要太重。
7. 匹配分数放在右上角，字体较大。
8. 优先级标签使用浅色胶囊标签。
9. “打开岗位页面”使用按钮样式，颜色可以使用蓝色。
10. 页面最大宽度控制在 1100px 到 1200px 左右，居中显示。
11. 移动端下所有信息块自动单列展示。
12. 不要使用深色大背景。
13. 不要使用复杂动画。
14. 不要使用太多颜色，整体保持浅蓝、白色、深蓝灰文字为主。

---

## 十、内容生成规则

生成岗位内容时必须遵守：

1. 不编造岗位。
2. 不编造原始投递链接。
3. 不编造薪资、福利、HR 回复率、录用概率。
4. 不要用普通国内企业岗位填充外企岗位列表。
5. 可以使用招聘平台上的公开岗位，例如 LinkedIn、猎聘、前程无忧、智联招聘、实习僧、BOSS 直聘等。
6. 岗位不一定只来自公司官网，但必须公开可验证。
7. 如果岗位来自第三方招聘平台，不需要在网页上展示“来源平台”字段。
8. 如果第三方平台岗位存在真实性或雇佣主体风险，应写进该岗位卡片的“风险提醒”里。
9. 如果没有原始链接，不要展示该岗位。
10. 如果岗位信息不完整，不要用编造内容补齐。

---

## 十一、推荐理由写法

推荐理由必须具体说明岗位为什么适合用户。
不要写空泛表达。

不要写：

“该岗位和用户背景匹配。”

应写：

“该岗位需要数据整理、业务分析和跨部门沟通能力，用户简历中已有 Excel 数据处理、商业分析项目和英文沟通经历，因此具备较好的基础匹配度。”

推荐理由建议控制在 1 到 3 句话，不要太长。

---

## 十二、风险提醒写法

风险提醒必须具体指出投递前需要核验的点。

可以包括：

1. 岗位是否仍在招聘。
2. 是否为公司直招。
3. 是否存在第三方外包或代理招聘。
4. 到岗时间和每周出勤是否匹配。
5. 岗位要求的工具或语言能力是否超出当前简历体现。
6. 是否要求长期实习。
7. 是否需要英文简历。
8. 是否对专业背景有硬性限制。

不要只写：

“注意岗位真实性。”

应写：

“该岗位来自第三方招聘平台，投递前建议确认是否为公司直招，并核验实际雇佣主体、办公地点和实习周期。”

---

## 十三、简历优化建议写法

简历优化建议必须具体到可以直接修改简历。

不要写：

“优化数据分析经历。”

应写：

“将项目经历中的‘负责数据分析’改为‘使用 Excel / Python 清洗并分析用户数据，提取转化率、留存率或销售趋势，并将结果整理成可视化报告，用于支持运营决策’。”

每个岗位卡片内的简历优化建议建议控制在 1 到 3 条。
如果空间有限，可以写成一段话，但必须具体、可执行。

---

## 十四、最终交付格式

最终请输出一个完整的单文件 HTML 页面。
HTML、CSS、必要的 JavaScript 都写在同一个文件里。

必须包含：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>外企求职报告</title>
  <style>
    /* CSS here */
  </style>
</head>
<body>
  <!-- Page content here -->
</body>
</html>
```

不要依赖外部 CSS 框架。
不要依赖外部图片。
不要依赖外部 CDN。
页面应可以直接保存为 `.html` 文件并在浏览器中打开。

---

1. 页面顶部只有“外企求职报告”一个标题。
2. 页面主体只有 Top 岗位推荐看板。
3. 岗位卡片中不再出现“来源平台”。
4. 岗位卡片中不再出现“推荐投递渠道”。
5. 岗位卡片中不再出现“发布/更新时间”。
6. 岗位卡片中不再出现“岗位匹配点”。
7. 岗位卡片中不再出现“主要差距”。
8. 岗位卡片中不再出现“下一步”。
9. 页面底部不再出现独立的“岗位风险提醒”板块。
10. 页面底部不再出现独立的“简历优化建议”板块。
11. 每个岗位卡片仍然保留推荐理由、风险提醒和简历优化建议。
12. 页面整体保持浅蓝、白色、简洁、现代的视觉风格。
