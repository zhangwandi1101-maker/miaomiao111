## Overview

1.1 目标任务：在已接入或已整理的外企岗位数据源中检索岗位信息，依照求职者的简历进行个性化推荐并给出推荐理由和简历修改意见，提供原始投递链接或推荐投递渠道。

1.2 使用对象：

1.  还未毕业正在找外企实习的学生

2.  即将毕业、希望投递外企校招或初级岗位的毕业生

3.  已经有工作经验想跳槽进外企的职场人

4.  已经毕业但仍有应届生身份的毕业生

1.3 典型应用场景：

1.  想海投外企，只知道去国内主流的招聘平台海投，hr回复慢、效率低下并且可能遇到第三方外包岗位

2.  即将毕业面临找工作的学生，非常迷茫，不知道是否要从事本专业的工作，想同时投递外企，给自己增加机会

3.  在一些社媒那里听博主的介绍，建议投递的外企平台，要一个个自己去找，时间成本大、效率低下

1.4 最终应交付的结果

本 Skill 面向“简历优先”的外企求职场景。用户上传简历后，助手需要先从简历中提取候选人画像，包括教育背景、专业、毕业时间、语言能力、实习经历、项目经历、技能关键词和潜在适配岗位方向。

随后，助手应基于用户补充的目标城市、偏好行业、目标岗位和到岗时间，从已接入、已整理或通过 Web Search 可验证的公开外企岗位信息中检索机会，并按照岗位匹配度、岗位新鲜度、来源可信度、信息完整度、投递路径效率和行业匹配度进行评分排序。

最终输出应包括候选人画像摘要、Top 岗位推荐、匹配评分、推荐投递渠道、推荐理由、风险提醒、简历优化建议和原始投递链接。

本 Skill 不用于提供泛泛的职业建议，也不应在没有可靠来源的情况下编造岗位、薪资、福利、HR 活跃度或回复率。

1.5 Skill 不是什么

Skill 不是根据用户简历只给出几条推荐的岗位，没有投递链接。

## When to use

当用户希望基于自己的简历寻找、评估或排序外企岗位机会时，使用本 Skill。或者说当用户指明要用喵喵求职工具的时候，用本 Skill

适合使用本 Skill 的情况包括但不限于：

（1）用户上传简历，希望判断自己适合投递哪些外企岗位

（2）用户想进外企，但不确定岗位方向

（3）用户希望寻找外企实习、校招、初级岗位或跳槽机会

（4）用户希望从外企官网、LinkedIn、猎聘、智联、实习平台或其他公开可验证来源中检索岗位机会

（5）用户希望根据简历匹配度、岗位新鲜度、来源可信度、信息完整度、投递路径效率和行业匹配度对岗位进行评分排序

（6）用户需要岗位推荐理由、风险提醒、简历优化建议和原始投递链接

## When not to use

当用户只需要与外企求职无关的泛泛职业建议时，不使用本 Skill。

以下任务不属于本 Skill：

（1）从零撰写一份通用简历，但不进行岗位匹配

（2）只做泛泛职业咨询，不基于简历分析岗位适配度

（3）承诺覆盖所有外企岗位或保证岗位实时有效

（4）在没有可靠来源的情况下判断 HR 活跃度、回复率、薪资福利或录用概率

（5）绕过登录、验证码、付费墙、平台限制或反爬机制获取岗位信息

（6）代替用户自动提交岗位申请

（7）提供法律、签证、劳动合同或移民判断

（8）在没有原始链接或置信度标注的情况下推荐岗位

## User context to collect

用户的主要输入是一份简历。助手应先从简历中提取候选人信息，并将简历视为用户背景信息的主要依据。

当简历信息不足以完成岗位匹配时，只补问推荐和排序所必需的缺失字段，不要一次性追问过多无关信息。

应从简历中提取或向用户补充收集以下信息：

学历层次、学校名称、专业名称、毕业时间、求职类型：实习、校招、初级岗位或有经验跳槽、目标城市、偏好行业、偏好岗位方向、实习经历、项目经历、比赛经历或校园经历、技能和工具、英语水平、其他语言能力、证书、到岗时间和每周可出勤天数、期望开始时间、工作地点或身份限制，如是否只能投中国大陆岗位、用户额外输入的偏好，例如公司类型、薪资期待或是否接受远程

如果某个字段无法从简历中识别，应标记为“未找到”，不要自行编造。只有当缺失字段会影响岗位筛选或排序时，才向用户追问。

## Core workflow

当用户上传简历并希望寻找外企岗位时，按照以下流程执行。

1. 接收并解析简历

- 接收用户上传的简历文件或粘贴的简历文本。

- 提取学历、学校、专业、毕业时间、语言能力、实习经历、项目经历、比赛经历、技能工具和岗位相关关键词。

- 将简历视为候选人背景信息的主要依据。

2. 构建候选人画像

- 将简历信息转换为结构化候选人画像。

- 推断用户可能适合的岗位方向，例如产品、运营、市场、咨询、数据分析、商业分析等。

- 对无法从简历中识别的信息标记为“未找到”，不要自行编造。

3. 补问缺失偏好

- 只询问岗位匹配所必需但简历中无法判断的信息。

- 常见缺失字段包括目标城市、求职类型、偏好行业、目标岗位方向、到岗时间、期望开始时间和是否接受远程。

4. 检索岗位机会

- 在已接入、已整理或通过 Web Search 可验证的公开外企岗位来源中检索岗位。

- 优先检索公司官网 Careers 页面，其次是 LinkedIn、猎聘、智联、实习平台和其他公开可验证来源。

- 保留原始链接和检索时间。

- 不绕过登录、验证码、付费墙或平台限制。

5. 提取并标准化岗位信息

- 提取公司名称、岗位名称、岗位类型、城市、平台来源、原始链接、岗位职责、任职要求、语言要求、出勤要求、薪资或补贴、发布时间和更新时间。

- 对不确定或缺失字段标记为“未知”。

- 不编造薪资、福利、HR 活跃度、回复率或录用概率。

6. 岗位去重与信息补全

- 当公司、岗位名称、城市、岗位类型和 JD 内容高度相似时，将不同平台上的同一岗位合并。

- 保留所有可用来源链接。

- 推荐最可靠的投递渠道，通常优先推荐公司官网。

7. 筛选与评分排序

- 先进行硬性筛选，再进行评分排序。

- 硬性筛选包括城市不匹配、求职类型不匹配、岗位已过期、缺少投递链接或到岗时间冲突。

- 对剩余岗位按照简历/JD 匹配度、岗位新鲜度、来源可信度、信息完整度、投递路径效率和行业匹配度评分。

- 按优先级排序。

8. 生成个性化投递清单

- 根据用户需求返回 Top 15 岗位推荐。

- 每个岗位应包含匹配分数、优先级标签、推荐投递渠道、原始投递链接、推荐理由、风险提醒和简历优化建议。

## Output format

- 对输出结果进行可视化，以网站形式呈现（生成网站前的内容必须读取并遵守 resources/website_skill_prompt.md，生成网站前的版式必须读取并遵守 resources/fronted_design.md）

- 如果用户明确要求更强的视觉表现、前端页面美化或更具设计感的网站报告，可额外参考 resources/fronted_design.md；但不得牺牲岗位真实性、来源可验证性和内容可读性。

- 候选人画像摘要：教育、专业、毕业时间、语言、经历亮点、适配岗位方向

- Top 岗位信息：公司、岗位、城市、匹配分、推荐渠道、推荐理由、风险提醒、简历优化建议、原始链接
